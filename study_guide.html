🧠 파이썬 코딩 이론 공부방

게임 뒤에 숨겨진 똑똑한 파이썬 개념들을 쉬운 설명으로 배워보아요!

💾 1. 세션 상태 (Session State)

"컴퓨터의 브레인 포스트잇 메모장"
Streamlit은 화면에서 버튼을 누르거나 글자를 입력할 때마다, 파일의 첫 줄부터 끝 줄까지 코드를 처음부터 다시 실행(새로고침)해요.
이때 일반 변수는 다 까먹고 초기화되지만, st.session_state에 넣어둔 값(예: 워들 정답 단어, 시도 횟수)은 컴퓨터가 포스트잇에 적어 기억하듯 잊지 않고 계속 보관해 둔답니다!

📦 2. 파이썬 바구니: 리스트(List)와 딕셔너리(Dictionary)

데이터를 이쁘게 담아두는 파이썬의 핵심 보관 상자예요:
- 리스트 (List): 물건을 순서대로 줄 세워 담는 상자예요. 예: w_guesses = ["APPLE", "HOUSE"] (사용자가 제출한 단어들을 순서대로 담을 때 사용)
- 딕셔너리 (Dictionary): '이름(Key): 값(Value)' 형태로 짝을 지어 서랍에 넣는 방식이에요. 예: {'A': 'green', 'B': 'gray'} (알파벳 키에 현재 어떤 색상인지 저장할 때 사용)

🎨 3. 워들 색상 판별 알고리즘 (그린 vs 옐로우)

제출한 단어가 정답과 얼마나 일치하는지 비교하는 3단계 논리에요:
- 🟩 초록색 (Green): 글자도 맞고, 위치도 정답과 완벽히 일치할 때!
- 🟨 노란색 (Yellow): 단어 안에 들어있는 글자이긴 하지만, 위치가 틀렸을 때!
- ⬛ 회색 (Gray): 단어 안에 아예 없는 글자일 때!

🚀 내 컴퓨터에서 게임 실행하기

수정한 게임 월드를 내 브라우저에서 켜보고 싶다면 터미널에 아래 명령어를 치면 돼요:

python -m streamlit run streamlit_app.py

💡 이 명령어를 치고 나면 브라우저 창이 열리면서 http://localhost:8501 주소에서 내 게임이 실행되는 것을 볼 수 있어요!

☁️ 깃허브(GitHub)에 내 코드 동기화하기

우리가 코드를 수정하고, 인터넷 깃허브로 보낼 때 사용한 4단계 흐름이에요:

  • 1
    파일 준비하기 (Stage)
    내가 수정한 파일들을 택배 상자에 담는 과정이에요.
    git add streamlit_app.py
  • 2
    포장하고 이름표 붙이기 (Commit)
    상자를 포장하고 어떤 걸 고쳤는지 메모지(이름표)를 붙여요.
    git commit -m "Update: Add Wordle game"
  • 3
    인터넷으로 트럭 발송하기 (Push)
    포장한 택배 상자를 인터넷 깃허브 저장소로 쏘아 올려요.
    git push -u origin main
